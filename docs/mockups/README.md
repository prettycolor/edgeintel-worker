# Phase 7 Mockups

These prototypes exist to align product direction before implementation.

## Files

- `phase-7-app-shell-prototype.html`
  - app-native control plane
  - provider settings live inside the workspace
  - local model onboarding appears as an in-app modal wizard
  - diagnostics stay in the app shell
- `phase-7-ui-prototype.html`
  - broader art-direction board
  - useful for tone and visual language
  - less representative of the final in-app control surface

## Recommended Direction

The app-shell prototype should drive implementation.

Why:

- EdgeIntel does not require a public-facing marketing website to deliver the Worker experience.
- The core product should be an authenticated application shell.
- Provider settings, tunnel orchestration, and connection tests belong in operational UI, not a landing page layout.
